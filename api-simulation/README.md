# API Simulation

This folder contains sample API simulations for payment transaction flows.

## Coverage
- Authorization (0100 → 0110)
- Reversal (0400 → 0410)

## Purpose
To demonstrate how transaction-level validation can be performed using API requests and responses, simulating real-world payment message flows.

## Tools
- Postman collection
- Sample request/response payloads

## Note
These are simplified simulations inspired by ISO8583-style messaging used in payment systems.
